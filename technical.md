# 정보의 격차를 줄이는 웹 접근성

코로나19의 장기화로 인해 온라인 쇼핑이 트렌드가 되었지만 시각장애인 A씨는 온라인 쇼핑 마저 제대로 할 수 없었다. 원하는 상품의 상세 정보를 확인하기 위해 '화면 낭독기'를 사용하였지만 대체 텍스트가 제대로 입력되지 않아 그저 '이미지'라는 말만 반복하였기 때문이었다.

시각장애인 B씨는 '국세청 홈택스'를 통해 집에서 간편하게 납세하고 싶었지만, 음성 지원을 이용하여 관련 항목을 선택했을 때 정확한 명칭이 아닌 엉뚱한 설명이 나와 이용에 어려움을 겪었다. 결국 A씨와 B씨는 서비스를 제대로 이용할 수 없었다. 왜 이런 일이 발생한 걸까?

## 웹 접근성?

A와 B씨가 서비스를 제대로 이용할 수 없었던 이유는 바로, ‘웹 접근성'이 제대로 지켜지지 않았기 때문이다.
웹 사이트, 도구, 기술이 장애를 가진 사람들이 비장애인과 동등한 서비스를 사용할 수 있도록 설계 및 개발된 것을 '웹 접근성(Web Accessibility)’이라고 한다. 웹 접근성은 「지능정보화기본법」에 따라 준수해야 하는 법적의무사항 이다.

웹은 기본적으로 하드웨어, 소프트웨어, 언어, 장소, 능력에 제약없이 모든 사람들이 사용할 수 있도록 설계되었다. 웹이 이 목표를 달성할 때 비로소 다양한 청력, 움직임, 시력, 인지 능력을 가진 사람들이 웹에 접근할 수 있다.

이와 같이 웹은 많은 사람들이 물리적 세상에서 의사소통과 상호작용하는 데에 마주하는 어려움을 웹이 없애주기 때문에 웹에서의 장애의 영향이 급격하게 변한다. 그러나 웹사이트, 어플리케이션, 기술이나 도구가 불충분하게 설계되면, 그것들은 사람들이 웹을 사용하는 것을 배제하는 장벽을 만들 수도 있다.
고품질의 웹사이트와 웹 도구를 만들고자 하며, 동시에 그들의 제품 및 서비스 사용에서 사람들을 배제하지 않으려는 개발자와 조직에게 웹 접근성은 필수적이다.

## 웹 접근성이 필요한 경우

웹 접근성이 필요한 장애를 4가지로 나눌 수 있다. 저시력 시각장애, 전맹 시각장애, 손 운동 장애, 중증 운동장애이다.

### 저시력 장애

저시력 시각 장애는 안경이나 콘택트렌즈, 약물치료나 수술적 처치에도 불구하고 일상적인 생활을 하는 데 어려움을 겪을 정도로 시력이 안 나오는 상태를 의미한다. 좋은 눈의 시력이 0.3 이하이거나 시야가 10도 이내로 줄어들어 있는 경우를 말하며, 빛도 감지하지 못하는 ‘완전 실명’의 경우는 제외한다. 환자들은 낮은 시력, 협소한 시야, 혹은 큰 암점을 가질 수도 있고 눈부심이나 비정상적인 색깔 인식 등으로 어려움을 겪는다.

따라서 웹 접근성 개선을 위해 명도대비, 컨텐츠 간의 구분, 색에 무관한 컨텐츠 인식, 명확한 지시 사항 제공, 조작 가능 등을 고려해야 한다. 비장애인도 안경을 벗으면 잘 보이지 않거나 어두운 빗길 운전시 표지판을 확인하기 착시현상으로 인해 색 구분이 어려웠던 경험 등에서 유사하게 겪을 수 있다.

### 전맹 시각 장애

전맹 시각 장애는 시력이 0으로 빛을 지각하지 못하는 시각장애를 말한다. 시력을 기준으로 0~0.02 미만까지는 맹, 0.02~0.4 미만까지는 준맹으로 구분한다. 다른 한편 1m 미만의 거리에서 지수를 인식하지 못하는 상태를 말한다.

따라서 전맹 시각 장애를 위한 웹 접근성을 개선하려면 음성과 점자가 출력되는 웹 환경이 필요하다. 그렇기에 스크린 리더를 고려하는 것이 가장 중요하다. 비장애인들도 유사한 경험을 할 수 있다. 밤에 방의 불이 다 꺼진 상태에서 핸드폰을 찾으려고 디바이스 인공지능 개인 비서 (ex, Siri, Bixby 등)를 사용한 경험, 방탈출 카페에 입장할 때 눈을 가리고 소리에만 의존해서 걸어갔던 경험 등이 있다.

### 손 운동 장애

손 운동 장애는 손을 자유롭게 사용할 수 없는 장애를 말한다. 대표적인 것으로는 손 떨림 장애와 한 손 장애가 있다. 손 떨림 장애는 40세 이상 인구의 약 4%에서 나타나는 가장 흔한 운동장애로, 연령이 증가하면서 발병률도 높아지는 신경퇴행성 질환이다. 일반적으로 소뇌-뇌간-시상-대뇌피질로 연결되는 운동기능 관련 신경회로가 비정상적으로 과항진되어 떨림이 나타난다. 한 손 장애는 말 그대로 사용 가능한 손이 하나만 있는 것을 말한다. 그렇기에 일반적인 키보드나 마우스 사용이 어렵다.

손 운동 장애를 위한 접근성 개선으로 키보드나 마우스 휠로 편리하게 사용할 수 있도록 키보드 사용성 보장, 초점 이동, 조작 가능, 응답 시간 조절, 사용자 요구에 따른 실행을 고려하는 것이 중요하다. 비장애인들은 손이나 팔을 다쳐 원활한 사용이 어려웠던 적, 마우스가 없거나 모바일 화면에 금이 가는 등의 장치에 고장이 발생하였던 경험, 짐이 많은 상태에서 한 손으로만 모바일을 조작해야 했던 경험 등을 통해 유사한 경험을 할 수 있다.

### 중증 운동 장애

중증 운동 장애 몸을 마음대로 움직일 수 없는 상태를 말한다. 면역체계의 이상으로 인해서 몸을 움직이기 위해 뇌가 명령한 신호가 운동신경에서 근육으로 잘 전달되지 않는다. 그렇기에 손을 이용하여 마우스나 키보드 사용이 어렵다.

그렇기 때문에 손 운동 장애와 마찬가지로 키보드 사용성 보장이 매우 중요한 접근성 개선이다. 비 장애인들이 유사하게 겪을 수 있는 경험은 설거지를 하고 있을 때 전화가 온 경우, 통화 버튼을 누르기 위해 손을 제외한 다른 신체부위를 사용하는 등이 있다.

## 웹 접근성 적용

1. 텍스트 컨텐츠와 배경간의 명도 대비

   웹페이지의 텍스트와 배경 사이에 충분한 명도 대비를 제공하는 것이 중요하다. 이는 저시력 시각장애인이 텍스트를 더 쉽게 읽을 수 있도록 돕는다. 일반적으로 WCAG(Web Content Accessibility Guidelines) 2.1 기준에 따르면 텍스트와 배경 간의 명도 대비는 최소 4.5:1 이상이어야 한다.

   ```html
   <style>
   	.high-contrast-text {
   		color: #000000; /* 검정색 텍스트 */
   		background-color: #ffffff; /* 흰색 배경 */
   	}
   </style>
   <p class="high-contrast-text">이 텍스트는 명도 대비가 높아 저시력 시각장애인이 쉽게 읽을 수 있습니다.</p>
   ```

2. 이웃한 컨텐츠 간의 구분

   이웃한 컨텐츠 간에 충분한 여백과 구분선을 사용하여 시각적으로 명확하게 구분하는 것이 필요하다. 이는 사용자가 각 컨텐츠를 독립적으로 인식하는 데 도움을 준다.

   ```html
   <style>
   	.content-section {
   		border-bottom: 2px solid #cccccc; /* 회색 구분선 */
   		margin-bottom: 20px; /* 하단 여백 */
   	}
   </style>
   <div class="content-section">
   	<p>첫 번째 컨텐츠 영역</p>
   </div>
   <div class="content-section">
   	<p>두 번째 컨텐츠 영역</p>
   </div>
   ```

3. 색에 무관한 컨텐츠 인식

   색상에 의존하지 않고도 정보를 전달할 수 있도록 해야 한다. 예를 들어, 중요한 정보는 색상뿐만 아니라 텍스트나 아이콘으로도 제공해야 한다.

   ```html
   <style>
   	.important-info {
   		color: red; /* 빨간색 텍스트 */
   		font-weight: bold; /* 굵은 텍스트 */
   		border-left: 5px solid red; /* 빨간색 좌측 경계선 */
   		padding-left: 10px; /* 좌측 여백 */
   	}
   </style>
   <p class="important-info">중요: 이 정보는 색상 외에도 텍스트와 경계선으로 강조되었습니다.</p>
   ```

4. 명확한 지시사항 제공

   사용자가 특정 작업을 수행할 때 명확한 지시사항을 제공해야 한다. 이는 특히 시각장애인이나 인지장애를 가진 사용자에게 도움이 된다.

   ```html
   <label for="email">이메일 주소를 입력하세요:</label>
   <input type="email" id="email" name="email" aria-describedby="email-help" />
   <small id="email-help">예: example@domain.com</small>
   ```

5. 사용자 입력 및 컨트롤 조작 가능

   웹페이지의 모든 입력 요소와 컨트롤은 쉽게 조작 가능해야 한다. 이는 손 운동 장애를 가진 사용자에게 특히 중요하다.

   ```html
   <button type="button">제출</button>
   ```

6. 키보드 사용성 보장

   웹페이지의 모든 기능은 키보드만으로도 접근 가능해야 한다. 이는 마우스를 사용할 수 없는 사용자를 위한 중요한 접근성 요구사항이다.

   ```html
   <a href="#main-content" class="skip-link">본문으로 건너뛰기</a>
   ```

7. 키보드에 의한 초점 이동

   키보드 사용자가 쉽게 초점을 이동할 수 있도록 해야 한다. 이는 특히 시각장애인이나 손 운동 장애를 가진 사용자에게 중요하다.

   ```html
   <style>
   	.focusable:focus {
   		outline: 3px solid #0000ff; /* 파란색 테두리 */
   	}
   </style>
   <button class="focusable">초점 가능한 버튼</button>
   ```

8. 시간제한 컨텐츠 응답 시간 조절

   시간 제한이 있는 컨텐츠는 사용자가 필요에 따라 시간을 조절할 수 있어야 한다. 이는 인지장애나 운동장애를 가진 사용자에게 특히 중요하다.

   ```html
   <p>이 퀴즈는 10분의 시간이 주어집니다. 필요 시 시간을 연장할 수 있습니다.</p>
   ```

9. 사용자가 의도하지 않는 기능에 대한 사용자 요구에 따른 실행

   사용자가 의도하지 않은 기능이 자동으로 실행되지 않도록 해야 한다. 이는 사용자가 웹페이지를 더 제어할 수 있도록 도와준다.

   ```html
   <input type="checkbox" id="enable-feature" name="enable-feature" /> <label for="enable-feature">기능 활성화</label>
   ```

10. 스크린 리더

    스크린 리더를 사용하는 사용자에게 정확한 정보를 제공하기 위해, 웹페이지의 모든 요소에 적절한 ARIA(Accessible Rich Internet Applications) 속성을 추가해야 한다.

    ```html
    <img src="image.jpg" alt="제품 사진" />
    <nav aria-label="주 메뉴">
    	<ul>
    		<li><a href="#home">홈</a></li>
    		<li><a href="#about">소개</a></li>
    		<li><a href="#contact">연락처</a></li>
    	</ul>
    </nav>
    ```

## 마무리 말

웹 접근성은 단순히 법적 의무를 넘어, 모든 사람이 정보와 서비스를 동등하게 누릴 수 있도록 하는 기본적인 인권의 문제이다. 장애인뿐만 아니라 비장애인도 다양한 상황에서 웹 접근성의 혜택을 받을 수 있다. 웹의 창시자 팀 버너스리가 말한 것처럼, 웹의 힘은 보편성에 있다. 모든 사람이 장애에 상관없이 웹에 접근할 수 있는 환경을 만드는 것이야말로 우리가 추구해야 할 목표이다.

웹 접근성을 고려한 웹사이트는 더 많은 사용자에게 다가갈 수 있고, 이는 결국 더 나은 사용자 경험을 제공한다. 개발자와 디자이너는 이러한 접근성을 통해 정보의 격차를 줄이고, 모두가 평등하게 웹을 사용할 수 있는 환경을 만들어야 한다. 이를 통해 더 포용적이고 공정한 디지털 세상을 만들어 나가는 것이 우리의 책임이다.
